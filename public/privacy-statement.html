Privacy Statement

Last Updated: January 20, 2025
Privacy Summary: The Datadog APM Azure Vending Machine is designed with privacy by design principles. We do not store your personal data, credentials, or application information.

1. Information We Collect

This service collects minimal information necessary for operation:

Authentication Data

  • Azure AD Identity: Your Azure Active Directory user ID and basic profile information (name, email) provided by Azure AD during authentication
  • Session Tokens: Temporary authentication tokens valid only during your session

Usage Data

  • Deployment Metadata: Information about ARM template deployments you initiate (timestamp, Azure subscription ID, resource group names)
  • Basic Analytics: Anonymized usage statistics to improve the service (page views, feature usage)
  • Error Logs: Technical error information to diagnose and fix issues (no personal data included)

2. Information We Do NOT Collect or Store

Privacy Protection: We explicitly do not collect or store sensitive information.

We do not collect, store, or have access to:

  • Azure Credentials: Your Azure passwords, access keys, or service principal credentials
  • Datadog API Keys: Your Datadog API keys or application keys
  • Application Data: Any data from your Azure App Services or applications
  • Business Data: Application metrics, logs, or performance data
  • Source Code: Any source code or configuration from your applications
  • Personal Files: Any files or data stored in your Azure resources

3. How We Use Information

The limited information we collect is used solely for:

  1. Authentication: Verifying your identity through Azure AD
  2. Authorization: Ensuring you have permission to deploy to your Azure subscription
  3. Service Operation: Facilitating ARM template deployments to your Azure resources
  4. Error Diagnosis: Troubleshooting technical issues and improving service reliability
  5. Usage Analytics: Understanding how the service is used to guide improvements (anonymized data only)

4. Data Storage and Security

Data Storage

  • Session Data: Stored temporarily in your browser's memory and cleared when you close the application
  • Authentication Tokens: Handled by Azure AD and not permanently stored by our service
  • Usage Logs: Stored temporarily for operational purposes and automatically purged

Security Measures

  • HTTPS Encryption: All data transmission uses TLS encryption
  • No Persistent Storage: No long-term storage of personal or credential data
  • Azure AD Integration: Leverages Microsoft's enterprise-grade authentication security
  • Minimal Data Principle: Only process the minimum data necessary for service operation

5. Third-Party Services

This service integrates with third-party platforms that have their own privacy policies:

Microsoft Azure

  • Purpose: Authentication, authorization, and resource deployment
  • Data Shared: User identity, subscription access permissions
  • Privacy Policy: Microsoft Privacy Statement

Datadog (Configuration Only)

  • Purpose: Configuring monitoring extensions (no direct data sharing)
  • Data Shared: None directly; API keys provided by you for configuration
  • Privacy Policy: Datadog Privacy Policy

GitHub Pages (Hosting)

  • Purpose: Hosting this web application
  • Data Shared: Basic web analytics (IP addresses, user agents)
  • Privacy Policy: GitHub Privacy Statement

6. Data Retention

We follow a minimal data retention policy:

  • Session Data: Automatically cleared when you log out or close the application
  • Authentication Tokens: Expire automatically according to Azure AD policies (typically 1-24 hours)
  • Usage Logs: Retained for up to 30 days for operational purposes, then automatically deleted
  • Error Logs: Retained for up to 90 days for troubleshooting, then automatically deleted

7. Your Rights and Choices

You have the following rights regarding your data:

  • Access: Request information about what limited data we have processed
  • Deletion: Request deletion of any usage logs (authentication data is managed by Azure AD)
  • Opt-out: Stop using the service at any time
  • Azure AD Controls: Manage your authentication data through your Azure AD administrator
Note: Since we don't store personal data long-term, most data deletion happens automatically when you log out.

8. Data Processing Legal Basis

We process your data based on:

  • Legitimate Interest: Providing the service you requested and improving service quality
  • Consent: Your explicit consent when using the service
  • Contract Performance: Necessary to perform the service you requested

9. International Data Transfers

Data processing occurs in:

  • GitHub Pages: Hosted on GitHub's global infrastructure
  • Azure Services: Within your selected Azure regions
  • Your Browser: Local processing in your web browser

All transfers comply with applicable data protection laws and use appropriate safeguards.

We may update this Privacy Statement occasionally to reflect changes in our practices or for legal compliance. We will:

  • Post the updated version on this page
  • Update the "Last Updated" date
  • For significant changes, provide notice through the application

Your continued use of the service after changes constitutes acceptance of the updated Privacy Statement.
